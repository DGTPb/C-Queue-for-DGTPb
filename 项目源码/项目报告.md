# 云山湖景区地图导航系统 — 项目报告

---

## 一、项目概述

### 1.1 项目背景

随着旅游业的快速发展，景区规模日益扩大，游客在游览过程中面临以下痛点：

- **寻路困难**：景区道路纵横交错，游客难以快速找到最优路线；
- **信息不对称**：缺少对景点分布、道路距离的全局感知；
- **游览效率低**：没有合理的游览顺序规划，容易走回头路。

传统纸质地图只能提供静态信息，无法进行路径计算和智能推荐。因此，本项目开发了一套**基于 Dijkstra 最短路径算法的景区导航寻路系统**，旨在为游客提供最短路径导航、游览路线推荐、景区地图可视化等一站式服务。

### 1.2 项目目标

本系统以 **C 语言**为开发语言，实现以下核心目标：

1. 建立景区景点与道路的结构化数据模型；
2. 实现基于 Dijkstra 算法的最短路径计算；
3. 提供基于贪心策略的游览路线推荐；
4. 实现 ASCII 字符画地图可视化；
5. 支持数据运行时热重载。

### 4.3 图结构体 `Graph`（核心）

定义在 [data.h](file:///c:/Users/qwert/Desktop/chenjx/src/data.h#L28-L35)：

```c
typedef struct {
    Spot spots[MAX_SPOTS];         // 景点数组，最多 64 个
    int  spot_count;
    Path paths[MAX_PATHS];         // 道路数组，最多 256 条
    int  path_count;
    int  adj[MAX_SPOTS][MAX_SPOTS]; // 邻接矩阵
    // adj[i][j]: 0=自身, -1=不连通, >0=距离(米)
} Graph;
```

**设计决策说明**：

采用**邻接矩阵**存储图结构而非邻接表的原因：

1. **空间开销可接受**：景区规模较小（≤64 景点），邻接矩阵仅需 64×64=4096 个 int，约 16KB；
2. **查询效率高**：查询两点间是否有边只需 O(1)，Dijkstra 松弛操作非常高效；
3. **实现简洁**：适合教学演示场景，代码可读性强。

## 五、算法设计与实现

### 5.1 Dijkstra 最短路径算法

**算法描述**：用于计算从起点到所有可达景点的最短路径。

**算法流程**（对应代码 [graph_dijkstra](file:///c:/Users/qwert/Desktop/chenjx/src/graph.c#L3-L39)）：

```
1. 初始化：
   - dist[] = -1（表示无穷大），起点 dist[src] = 0
   - prev[] = -1（无前驱）
   - visited[] = 0（未访问）

2. 循环 n 轮（n 为景点数）：
   a. 在未访问点中，贪心选出 dist 值最小的顶点 u
   b. 标记 u 为"已确定"（visited[u] = 1）
   c. 用 u 松弛所有邻居 v：
      若 dist[u] + adj[u][v] < dist[v]，则更新 dist[v] 和 prev[v]

3. 结束：dist[] 存储最短距离，prev[] 存储前驱链
```

**核心代码**：

```c
void graph_dijkstra(const Graph *g, int src_idx, int *dist, int *prev) {
    int n = g->spot_count;
    unsigned char visited[MAX_SPOTS];
    
    for (int i = 0; i < n; i++) {
        dist[i] = -1;      // -1 表示不可达（无穷大）
        prev[i] = -1;
        visited[i] = 0;
    }
    dist[src_idx] = 0;
    
    for (int iter = 0; iter < n; iter++) {
        // 选距离最小的未访问点
        int u = -1, best = -1;
        for (int i = 0; i < n; i++) {
            if (visited[i] || dist[i] < 0) continue;
            if (u < 0 || dist[i] < best) { u = i; best = dist[i]; }
        }
        if (u < 0) break;
        visited[u] = 1;
        
        // 松弛操作
        for (int v = 0; v < n; v++) {
            if (visited[v]) continue;
            int w = g->adj[u][v];
            if (w <= 0) continue;
            int nd = dist[u] + w;
            if (dist[v] < 0 || nd < dist[v]) {
                dist[v] = nd;
                prev[v] = u;
            }
        }
    }
}
```

**时间复杂度**：O(V²)，V 为顶点数。对于 V≤64 的景区图，性能完全满足实时交互需求。

**路径回溯**（对应代码 [graph_shortest_path](file:///c:/Users/qwert/Desktop/chenjx/src/graph.c#L41-L72)）：

```
从终点 dst 出发，沿 prev[] 反向追溯到起点 src，
用栈收集逆序路径，再反转为正序输出。
```

### 5.2 最近邻启发式游览路线推荐

**算法描述**：用于解决"从起点出发，游览所有目标景点"的 TSP（旅行商问题）近似求解。

**算法流程**（对应代码 [route_recommend](file:///c:/Users/qwert/Desktop/chenjx/src/route.c#L3-L56)）：

```
1. 从起点出发，构建待访问目标集合
2. 循环直到所有目标被访问：
   a. 对当前点运行 Dijkstra，计算到所有未访问目标的距离
   b. 选择距离最近的未访问目标作为下一站
   c. 更新当前点，标记该目标已访问
3. 计算总距离和预估时间
```

**核心代码**：

```c
RouteResult route_recommend(const Graph *g, int start_idx,
                            const int *targets, int target_count) {
    // ...初始化...
    
    while (remaining > 0) {
        graph_dijkstra(g, current, dist, prev);
        
        // 选最近的未访问目标
        int best = -1, bestd = -1;
        for (int i = 0; i < n; i++) {
            if (!want[i] || dist[i] < 0) continue;
            if (best < 0 || dist[i] < bestd) {
                best = i;
                bestd = dist[i];
            }
        }
        if (best < 0) { r.reachable = 0; break; }
        
        r.total_distance += bestd;
        r.visit_order[r.count++] = best;
        want[best] = 0;
        current = best;
        remaining--;
    }
    // ...
}
```

**特点**：贪心策略，虽然不是全局最优解，但在实际景区场景下效果良好，且计算速度快（O(k·V²)，k 为目标景点数）。

### 6.1 数据加载模块

**功能**：从文本文件解析景点和道路数据，构建邻接矩阵。

**实现要点**（对应代码 [data_load](file:///c:/Users/qwert/Desktop/chenjx/src/data.c#L73-L98)）：

1. **邻接矩阵初始化**：对角线为 0（自身），其余为 -1（不连通）；
2. **重边处理**：若两景点间有多条道路，自动保留距离最短的一条；
3. **无向图构建**：每条道路在邻接矩阵中双向赋值（`adj[ia][ib] = adj[ib][ia] = d`）。

## 九、技术亮点

### 9.1 用 `-1` 哨兵替代 `INF` 常量

标准 Dijkstra 通常用大整数 `INF` 表示"不可达"，本系统用 `-1` 作为哨兵值：

```c
if (dist[v] < 0 || nd < dist[v]) {   // dist[v]==-1 等价于 ∞
    dist[v] = nd;
    prev[v] = u;
}
```

**优势**：避免 `dist[u] + w` 与 `INF` 相加时的溢出风险，代码更安全。

### 9.2 跨平台终端适配

通过条件编译实现 Windows 和 POSIX 系统的终端适配，同一套代码可在不同平台正确显示中文和彩色输出。

### 9.3 ASCII 字符地图可视化

无需任何图形库，仅用字符在终端中渲染完整景区地图，支持路径高亮，直观展示导航结果。

### 9.4 数据与逻辑分离

景点和道路数据存储在外部文本文件中，支持运行时热重载，修改数据文件后无需重新编译即可生效。

### 9.5 重边自动取最短

构建邻接矩阵时，若两景点间有多条道路，自动保留距离最短的一条：

```c
if (g->adj[ia][ib] < 0 || d < g->adj[ia][ib]) {
    g->adj[ia][ib] = d;
    g->adj[ib][ia] = d;
}
```

## 十一、项目总结

本系统以**景区导航寻路**为核心需求，采用 **C 语言**实现，通过 **Dijkstra 最短路径算法**解决了景区内的最优路径规划问题，辅以**最近邻启发式算法**实现游览路线推荐。系统具有以下特点：

1. **模块化设计清晰**：按职责划分为数据层、算法层、展示层、交互层，各模块职责单一；
2. **数据结构合理**：采用邻接矩阵存储图结构，兼顾空间开销和查询效率；
3. **算法实现正确**：Dijkstra 算法和最近邻贪心算法均正确实现，经过手工验证；
4. **地图可视化直观**：ASCII 字符画地图清晰展示景区布局和导航路径；
5. **具备良好的实用性和可扩展性**：支持数据热重载，景点和道路数量可扩展。

**未来改进方向**：
- 引入优先队列优化 Dijkstra 算法，降低时间复杂度；
- 增加更多道路属性（如坡度、拥挤度），实现加权路径规划；
- 扩展为 GUI 界面，提升用户体验；
- 增加多语言支持。
